# v130：任务执行结果摘要采集第一阶段实施计划

> 日期：2026-06-03
> 执行者：Codex
> 说明：本文承接 v129 任务状态自动采集第一阶段，继续深化 Jarvis Lite 1.0 验收线中的任务状态和失败复盘。

## 目标

- 当前任务运行中执行显式 slash command 时，在命令返回后把紧凑结果摘要写回最近匹配的任务事件。
- 失败复盘中的“自动采集上下文”展示命令输入、路由详情和结果摘要，便于判断失败前的上一条命令是否已经成功。
- 结果摘要只保留压缩后的短文本，避免把长输出、完整文档正文或大段上下文写入任务事件。
- `/task-fail` 自身和其他任务管理命令继续不进入自动采集事件。

## 非目标

- 不自动截图、不自动 OCR、不自动重新执行外部动作。
- 不自动重放命令，不把失败复盘转成任务编排器。
- 不采集真实桌面内容、联系人、凭据或全文输出。
- 不重构 `JarvisAgent.handle()` 的全部返回路径；本阶段只覆盖显式 slash command 分支。
- 不把自然语言或 LLM 建议自动映射为键鼠、窗口或应用动作。

## 实施步骤

1. 扩展 `tests/test_task_state.py`，先验证任务事件结果摘要会写回最近匹配事件，并在失败复盘中展示。
2. 扩展 `tests/test_agent.py`，验证当前任务中执行 `/dir-add` 后，`/task-fail` 的自动采集上下文展示命令执行结果摘要。
3. 将版本元数据测试期望提升到 `0.125.0`，验证 RED。
4. 扩展 `src/jarvis_lite/task_state.py`，新增任务事件结果写回函数和结果压缩逻辑。
5. 在 `JarvisAgent` 显式 slash command 分支调用任务事件结果写回，并保持任务管理命令过滤边界。
6. 同步 `pyproject.toml`、`src/jarvis_lite/__init__.py`、README、PROJECT-PLAN、方案索引、文档索引、进度与验证记录。
7. 运行目标测试、相邻回归、全量 unittest、临时项目 smoke、桌面 smoke、打包后 smoke 和静态检查。

## 验收标准

- 当前任务运行中执行 `/dir-add 工作区 路径` 后，随后 `/task-fail 原因` 的“自动采集上下文”包含 `结果：已登记常用目录：工作区 -> 路径`。
- 结果摘要写回最近匹配的任务事件，并随失败记录持久化。
- `/task-fail` 自身不进入自动采集上下文。
- 当前任务已经失败后，不再把 `/task-fail` 的返回结果写回任务事件。
- 输出继续明确“不自动截图、不自动 OCR、不自动重新执行外部动作”。
