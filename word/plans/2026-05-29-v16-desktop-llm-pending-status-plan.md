# Jarvis Lite v16：桌面外脑待补充状态方案

> 日期：2026-05-29
> 执行者：Codex
> 说明：本文承接 v15 LLM 外脑澄清轮数与过期策略，明确 `0.12.0` 的桌面可观察性收口。

## 核心结论

`0.12.0` 不继续扩展自然语言规则，而是把已经存在的 LLM pending 澄清状态放到桌面面板固定区域：

```text
LLM 外脑需要补充
  -> Agent 保存 pending
  -> Bridge 读取只读状态快照
  -> Panel 固定展示待补充问题和轮次
  -> 用户补充或取消后刷新状态
```

这样用户不用点“最近上下文”也能知道当前桌面助手是否正在等待外脑补充。

## 设计边界

- 不新增自然语言正则模板。
- 不调用 `/recent-context` 做后台状态刷新，避免污染对话历史。
- 不改变 LLM provider adapter。
- 不改变 InnerBrain 缺槽澄清逻辑。
- 只新增桌面面板的状态读取和展示。

## 0.12.0 收口内容

- Agent 暴露只读 LLM 澄清状态文本。
- DesktopBridge 在每次发送后附带最新 pending 状态。
- AssistantPanel 增加固定外脑状态 label，启动时、发送后、取消后都会刷新。
- 有 pending 时显示澄清问题、轮次和取消提示。
- 无 pending 时显示“外脑待补充：无”。
- 项目版本同步到 `0.12.0`，README、PROJECT-PLAN、验证记录和安装包同步本轮范围。

## 后续方向

- 根据真实使用反馈，把外脑状态进一步拆为“配置状态”“待补充状态”“最近调用结果”三个轻量状态块。
- 继续打磨 LLM prompt 和联网搜索来源总结质量，而不是扩大正则模板。
