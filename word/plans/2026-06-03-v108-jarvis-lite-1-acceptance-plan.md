# v108：Jarvis Lite 1.0 验收线方案

> 日期：2026-06-03
> 执行者：Codex
> 说明：本文承接 v107 PROJECT-PLAN 主干同步，明确 Jarvis Lite 从 InnerBrain 本机 evaluation 微闭环转入 1.0 验收路线。

## 背景

`0.103.0` 已经完成 PC Agent、InnerBrain、LLM 外脑、联网搜索、桌面入口、Windows 安装包和 InnerBrain 本机 evaluation 治理的基础闭环。版本号虽然已经到 `0.103.0`，但该编号主要来自大量小粒度闭环，不代表产品已经接近正式 `1.0.0`。

下一阶段需要先定义 `1.0` 的验收线，避免继续把方案版本用于局部文案、空状态或报告入口微调。`1.0` 的目标应从“内脑评估可观察”提升为“私人 PC 助手能听懂自然语言并真实操作电脑和常用应用”。

## 1.0 定位

Jarvis Lite 1.0 是本地优先的个人 PC Agent 正式第一代版本。它的核心不是让用户记住命令，而是让用户用中文、英文或中英混合的自然语言表达真实电脑任务，由 Jarvis Lite 完成理解、授权判断、桌面/应用操作、结果反馈和失败复盘。

1.0 的一句话验收标准：

> 用户用自然语言说一件常见电脑任务，Jarvis Lite 能理解、判断是否可以直接执行、操作目标应用完成任务；如果失败，能说明卡在哪一步，并从日常表达里逐步学习用户的术语、别名、路径、联系人和偏好。

## P0 验收项

### 1. 自然语言主入口

- 支持中文、英文和中英混合表达。
- 高频任务不要求用户记忆 `/command`。
- InnerBrain 优先处理高频本地工作流；复杂、开放或低置信输入进入 LLM 外脑 fallback。
- 低置信或槽位缺失时追问，不直接猜测执行。
- 支持个人术语和别名逐步沉淀，例如“代理面板”指 Clash Verge，“这个项目”指当前 Jarvis Lite 项目。

### 2. 意图授权层

- 在自然语言理解之后、桌面动作执行之前增加授权判断。
- 明确区分 `询问`、`准备`、`执行` 和 `建议`。
- 判断依据包括动作风险、用户授权明确度、缺失槽位、个人免确认规则和识别置信度。
- 输出结果限定为：直接执行、准备后确认、追问补充、拒绝或降级。
- 用户明确要求最终动作且对象、内容、渠道明确时，不重复询问；准备型、判断型或建议型表达只准备不提交。

### 3. 自动记忆与配置管家

- 配置文件、样本文件、联系人别名、常用路径、应用路径和免确认规则是内部存储，不要求用户手工维护。
- 日常入口以自然语言为主，用户可以说“记一下”“以后我说 A 就是 B”“忘掉刚才那条规则”。
- 明确保存指令优先级最高；内容清楚且风险低时直接保存。
- 日常对话中出现的表达先进入日志或候选池，不无脑写入长期配置。
- 重复出现、置信度高、保存后能减少打扰的候选，可以主动询问是否固化为记忆或配置。
- 每条长期记忆、配置和授权规则需要记录来源、时间、用途和可撤销入口。

### 4. 应用注册与窗口感知

- 首批常用应用固定为 Chrome、QQ、微信、IntelliJ IDEA 和 Clash Verge。
- Jarvis Lite 能自然语言打开、切换、识别这些应用是否运行。
- 能获取当前前台窗口标题、进程、窗口列表和基础状态。
- 能截图当前屏幕或目标窗口。
- 能基于窗口标题、截图和 OCR 判断当前界面是否处于可操作状态。

### 5. 键鼠自动化基础层

- 支持打开应用、切换窗口、发送快捷键、输入文本、点击目标、保存截图。
- 桌面动作必须可被任务计划调用，而不是散落在应用特定逻辑里。
- 动作执行结果需要返回成功、失败、超时、找不到目标或需要用户介入。

### 6. 五个常用应用工作流

- Chrome：打开网页、搜索、截图、提取页面信息、保存资料。
- Clash Verge：打开面板、识别代理状态、识别当前节点或模式、切换前默认确认。
- IntelliJ IDEA：打开应用和项目、识别当前项目状态，后续再扩展运行测试或打开终端。
- QQ：打开应用、判断登录状态、查找联系人、准备消息；明确授权且命中规则时发送。
- 微信：打开应用、判断登录状态、查找联系人、准备消息；明确授权且命中规则时发送。

### 7. 任务状态与失败复盘

- 多步骤任务必须记录当前步骤、已完成步骤、失败步骤和下一步建议。
- 任务中断后可以继续、取消或重新执行。
- 失败记录至少包含用户原话、解析意图、授权判断、当前应用/窗口、截图或 OCR 文本、执行步骤、失败原因和下一步建议。
- 失败记录可反向生成 InnerBrain evaluation 样本、候选记忆或修复建议。

## P1 候选项

- 语音输入：PC 端先支持按键说话或托盘录音，多端语音入口放到 1.1 以后。
- 文档生成与处理：优先覆盖 Markdown、docx、xlsx、PPT 的打开、读取、整理和保存。
- 网页资料采集：自动打开网页、截图、OCR、提取正文、保存进知识库。
- 外脑/联网搜索产品化：继续增强 provider 配置、失败恢复、来源展示、用量观察和总结质量。
- 桌面控制台：展示当前任务、当前应用、最近动作、候选记忆、待确认事项和失败记录。
- 授权规则可视化：自然语言管理为主，界面只负责查看和手动修正。
- 资料库长期维护：自动归档、打标签、发现重复、生成日报或周报、提醒未处理事项。

## 记忆与配置边界

### 可以自动保存

- 最近输入、路由结果、执行结果和失败记录。
- 非敏感使用习惯候选。
- 已成功执行过且低风险的自然语言样本候选。
- 临时上下文，例如当前项目、刚才的文件、最近打开的窗口。

### 用户明确说了就可保存

- 应用别名、常用目录、常用网站、默认浏览器、项目简称、个人术语和非敏感偏好。
- 示例：“以后我说代理面板，就是 Clash Verge。”
- 示例：“这个目录记成 Jarvis 项目。”

### 需要确认后保存

- 联系人别名。
- 免确认规则。
- 自动发送消息规则。
- 涉及 QQ、微信、账号、密码、API key、token 或其他敏感资料的内容。
- 任何会长期影响自动执行外部动作的配置。
- 任何仅由系统推断得出，但保存后会改变长期行为的规则。

### 只临时记录

- 假设句、询问句、一次性任务和对象不清楚的表达。
- LLM 或联网搜索推断出的事实，除非用户确认。
- 用户没有表达保存意图、也没有达到重复高置信候选条件的普通聊天内容。

## 旧方案整理结论

- v1-v6 仍是路线基础：PC Agent、本地内脑、LLM 外脑和联网搜索的分工继续有效。
- v7-v22 形成外脑配置、联网搜索、路由历史和 InnerBrain 候选基础，可作为 1.0 的自然语言与外脑支撑能力。
- v23-v39 形成候选训练、样本写入和本机 evaluation 基础，后续应从“人工显式补样本”为主，升级到“日常对话自动候选 + 用户确认固化”。
- v40-v107 大多是 InnerBrain 本机 evaluation 的报告、文件聚焦、待处理失败和空状态治理，保留为历史版本，不再作为下一阶段主线继续细分。
- `word/plans/README.md` 的“当前方案”应从 v104 校正为本 v108，避免索引落后于实际最新方案。
- `word/PROJECT-PLAN.md` 的“下一阶段”应从继续观察 `runtime.jsonl` 失败报告，调整为推进 1.0 P0 验收项。

## 非目标

- 不在 1.0 同时推进手机端、手表、车机或 AR 眼镜入口。
- 不把所有应用都纳入自动化；首批只覆盖 Chrome、QQ、微信、IDEA 和 Clash Verge。
- 不要求用户提前录入全部常用说法。
- 不把每一句日常聊天都自动写入长期记忆或训练样本。
- 不让 README 承担完整方案和验收清单，README 仍只做项目介绍和快速启动。

## 推荐落地顺序

1. 应用注册与窗口感知。
2. 截图与 OCR。
3. 键鼠自动化基础层。
4. 意图授权层。
5. 自动记忆与配置管家。
6. Chrome 和 Clash Verge 低风险工作流。
7. QQ、微信准备式工作流。
8. IDEA 项目打开与状态识别。
9. 任务状态、中断恢复和失败复盘。

## 验收方式

- 每个 P0 能力必须有本地自动化测试或可重复的本地 smoke。
- 自然语言能力需要固定 seed 样本、本机 evaluation 样本和真实失败样本回归。
- 桌面自动化能力需要在无外部 CI 的本机环境执行截图/OCR/窗口状态 smoke。
- 涉及发送消息、联系人、凭据和免确认规则的功能必须验证授权边界、缺失槽位追问和撤销路径。
- 正式 `1.0.0` 前需要一份独立 1.0 验收报告，逐项标注通过、暂缓或降级。
