# Jarvis Lite 桌面更新检查第一版设计

> 日期：2026-05-20
> 执行者：Codex

## 目标

为 Jarvis Lite 增加低风险的更新检查入口，让用户可以看到当前版本、是否有新版本、下载地址和更新说明。该阶段先解决“知道有更新并能拿到安装包”的体验，不做静默自动更新。

## 范围

- 新增更新清单解析能力，清单使用 JSON：
  - `version`：最新版本号。
  - `download_url`：安装包下载地址。
  - `release_notes`：更新说明，可选。
  - `published_at`：发布时间，可选。
- 支持从本地文件路径或 `http/https` URL 读取更新清单。
- 默认通过环境变量 `JARVIS_LITE_UPDATE_MANIFEST_URL` 指定更新源。
- 新增 `/update-status [清单路径或URL]` 命令。
- 桌面面板和托盘增加“检查更新”快捷命令。
- 更新检查只输出提示和下载入口，不自动下载、不自动安装。

## 设计边界

第一版不做后台轮询，也不自动替换正在运行的应用。当前 IExpress 安装器已经支持覆盖安装前关闭运行中的 exe，但自动下载、校验、回滚和静默安装会带来更多失败路径，应该放到后续专业安装器阶段。

更新源未配置时，命令会明确提示如何配置，而不是静默失败。

## 不做事项

- 不接入摄像头、麦克风或真实语音识别。
- 不做自动下载。
- 不做静默自动安装。
- 不替换安装器。
- 不做代码签名。

## 验证标准

- 版本比较能正确处理 `0.10.0` 大于 `0.2.0`。
- 无更新源时返回当前版本和配置提示。
- 本地 JSON 清单有更高版本时返回下载地址和更新说明。
- 当前版本已是最新时返回无需更新。
- `/update-status` 命令可使用本地清单路径完成检查。
- 桌面快捷命令包含“检查更新”。
- 全量本地测试、桌面 smoke、安装器构建和打包 exe smoke 通过。
