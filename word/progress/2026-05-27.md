  - 解析 provider 返回的 token usage，并把 `input_tokens`、`output_tokens`、`total_tokens` 记录到 `logs/jarvis.log`。
- 继续补充 LLM 配置诊断：
  - `/llm-status` 会列出缺少的模型、API key、兼容端点 base URL 和未知 provider。
  - 诊断只显示缺失项，不打印 API key 内容。
- 继续补充 LLM 本地用量辅助：
  - 新增 `/llm-usage`，从本地 `logs/jarvis.log` 汇总 provider/model 维度的调用次数和 token 用量。
  - 用量汇总不触网，不需要真实 API key。
- 继续补充 LLM 配置辅助：
  - 新增 `/llm-config-example [provider]`，输出 PowerShell 环境变量模板。
  - `qwen` / `gemini` 先映射到 OpenAI-compatible 端点模板，原生 adapter 后续接入。

  - OpenAI-compatible provider 支持用户直接粘贴完整 `/v1/responses` URL，调用 SDK 时自动归一化为 `/v1` base URL。
  - `/llm-status` 会在发生归一化时展示 SDK 实际使用的 Base URL。
  - `/llm-config-example openai-compatible` 会提示兼容端点可填写 `/v1` base URL 或完整 `/v1/responses` URL。
