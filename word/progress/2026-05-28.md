## 完成事项

- 增强 LLM fallback 近期上下文：
  - `JarvisAgent._llm_context_lines()` 继续提供记忆摘要、最近资料和最近目录。
  - 复用已有 `suggest_next_actions_from_context()`，在有近期上下文或经验记忆时追加 `下一步建议：...`。
  - fake provider 测试覆盖 LLM fallback 能收到 `继续处理最近资料：/read note.txt；/tag note.txt 标签...`。
- 继续增强 LLM fallback 的资料上下文：
  - 知识库问答命中多份资料后，LLM context 会追加 `最近搜索结果：N 条`。
  - 最多列出 3 条 `data/<path>`，方便模型建议读取或标注具体资料。
  - fake provider 测试覆盖 `/ask` 后的最近搜索结果会进入 LLM context。
- 保持既有边界：
  - 本地命令、身份、本地自然语言意图和知识库问答仍优先于 LLM。
  - LLM 仍只返回结构化意图，命令执行仍由 `JarvisAgent` 控制。
  - 本轮不新增 provider，不触发真实网络调用，不写入真实 API key。
- 收口 LLM 调用前测试能力和执行边界：
  - 新增 `/llm-context-preview`，可直接查看 LLM fallback 会携带的记忆摘要、最近资料、最近搜索结果和下一步建议，不调用 provider。
  - `OpenAIResponsesProvider` 的命令白名单提取为共享常量，Agent 执行 LLM command intent 前会再次校验未知命令。
  - `/llm-status` 展示 API key 是否已配置、当前配置是否会触发网络调用，以及完整 `/v1/responses` URL 归一化后的 SDK Base URL。
  - OpenAI provider 401、429、超时等错误会转为更可读的中文提示，并避免把 API key 写进错误文本。
- 准备新版安装包：
  - 项目版本提升到 `0.1.1`。
  - 安装脚本覆盖安装时会提示旧程序文件已覆盖，并明确用户数据保留在 `%LOCALAPPDATA%\Jarvis Lite`。
  - 已安装旧版时默认走覆盖安装；只有安装目录、快捷方式或卸载注册表异常时才建议卸载重装。
- 根据用户安装后的真实聊天日志修复自然语言识别缺口：
  - 确认 `/help` 已出现 `/llm-context-preview`，说明 `0.1.1` 已经生效，根因不是安装失败，而是本地自然语言意图层缺少对应规则。
  - 新增问候识别：`早上好`、`你好` 等不会再落到“已读取长期记忆”的兜底回复。
  - 新增助手身份询问：`你叫什么名字` 会直接回答 `我叫 Jarvis Lite`。
  - 新增桌面快捷方式删除：`帮我把桌面上的比特云手机和比特浏览器的快捷方式删掉` 会只删除桌面目录下明确点名的 `.lnk` 快捷方式，并报告未找到项。
  - 安装完成弹窗和安装脚本输出加入版本号，项目版本提升到 `0.1.2`，便于用户确认新安装包已经生效。
- 将自然语言回复和思考行为讨论整理为 v4 双脑方案：
  - 新增 [../plans/2026-05-28-v4-inner-brain-llm-dual-brain-plan.md](../plans/2026-05-28-v4-inner-brain-llm-dual-brain-plan.md)。
  - 明确不继续把用户所有说法堆成正则，也不从零训练通用小型 LLM。
  - 确定 InnerBrain 本地内脑职责：意图识别、槽位抽取、置信度、缺失信息、执行/追问/交给外脑的策略判断。
  - 确定 LLM 外脑职责：复杂理解、总结、规划、自然表达和训练样本扩展。
  - 明确训练数据应来自 `text -> intent -> slots` 标注样本、用户真实日志、现有命令能力和 LLM 生成的结构化表达变体，不直接从搜索引擎抓无标签语料训练。
  - 更新 `word/PROJECT-PLAN.md`、`word/plans/README.md` 和 `word/文档索引.md`，将当前路线调整为 PC Agent -> InnerBrain 内脑 -> LLM 外脑 -> 多端入口。
- 落地 InnerBrain v1：
  - 新增 `src/jarvis_lite/inner_brain.py`，定义 `InnerBrainResult`、`InnerBrainTrainingSample` 和 `InnerBrainPolicy`。
  - `InnerBrain` 先包装既有 `parse_natural_language_intent()` 为 `source=legacy_rule`，保留旧行为。
  - 内置 seed 样本支持 `麻烦看一下知识库摘要` 和 `把桌面比特浏览器快捷方式删除` 等表达，不再把每个说法都写成 Agent 正则。
  - 支持运行态 `data/inner-brain/training/*.jsonl` 样本，格式为 `text -> intent -> slots`，后续可沉淀用户真实日志。
  - `JarvisAgent` 在 data 问答和 LLM fallback 之前调用 InnerBrain；高置信度执行，中置信度澄清，低置信度继续交给 data/LLM fallback。
- 增加 InnerBrain 可观察调试入口：
  - 新增 `/inner-brain-status`，展示 legacy 规则启用状态、seed/runtime 样本数量、置信度阈值和训练样本目录。
  - 新增 `/inner-brain-preview 文本`，只预览策略、意图、来源、置信度、槽位和候选命令，不执行任何本地动作。
  - 新增回归测试确认 preview 桌面快捷方式删除表达时不会删除临时 Desktop 下的 `.lnk`。
- 增加 InnerBrain runtime 样本采纳闭环：
  - 新增 `/inner-brain-adopt 文本`，把当前 InnerBrain 可识别的非 `unknown` 结果保存到 `data/inner-brain/training/runtime.jsonl`。
  - 保存时会把自然语言意图中的有效槽位一并落盘，例如桌面快捷方式删除表达会保存 `items`。
  - 重复样本不重复写入，保存动作只写训练样本，不执行识别出的命令或本地动作。
- 增加 InnerBrain 人工标注入口：
  - 新增 `/inner-brain-label 文本 => intent [slot=value ...]`，用于把 `unknown` 或误识别表达人工标注为 `text -> intent -> slots` 样本。
  - 支持 `command=/kb`、`items=比特浏览器`、`tags=项目,归档`、`missing=source` 等参数，保存后当前 Agent 会立即刷新 InnerBrain。
  - 标注命令只写 runtime 样本，不执行被标注出的命令或桌面动作。
- 增加 InnerBrain 口语化教学入口：
  - 新增 `/inner-brain-teach 文本 => /命令`，把自然语言短句绑定到已知命令，并写入 `data/inner-brain/training/runtime.jsonl`。
  - 支持“以后我说“文本”就是 /命令”“把“文本”记成 /命令”等口语教学句式。
  - 教学动作只保存样本并刷新当前 Agent，不执行目标命令；未知命令不会写入训练样本。
- 准备 InnerBrain `0.1.3` 可安装测试版：
  - 项目版本提升到 `0.1.3`。
  - 本轮安装包包含 InnerBrain v1、preview/status、runtime 样本采纳、人工标注和口语教学入口。
  - 安装器仍走覆盖安装策略，保留 `%LOCALAPPDATA%\Jarvis Lite` 用户数据。
- 补齐 LLM 外脑本地配置入口：
  - 新增 `config/llm.example.json` 模板，并将真实 `config/llm.local.json` 加入 `.gitignore`。
  - `LLMSettings.from_sources()` 支持读取 `config/llm.local.json`，环境变量可覆盖同名字段。
  - `JarvisAgent` 初始化时按当前 `ProjectPaths` 构建 LLM Router；安装版会使用 `%LOCALAPPDATA%\Jarvis Lite\config\llm.local.json`。
  - 新增 `/llm-enable`，用户说“开启外脑”会走同一入口，展示配置路径、状态和 smoke 命令，并在运行中重新加载配置。
