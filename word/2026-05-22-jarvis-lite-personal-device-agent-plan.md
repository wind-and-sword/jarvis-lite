# Jarvis Lite 个人设备级 Agent 融合方案

> 日期：2026-05-22  
> 执行者：Codex  
> 说明：本文基于当前 Jarvis Lite 进度和本轮 Agent 技术讨论，重新整理项目后续方向。

## 1. 新定位

Jarvis Lite 不只是本地聊天助手，而是个人设备级 Agent 的起点。

一句话定位：

> Jarvis Lite 是一个让 AI 逐步学会使用电脑、手机、手表和个人数据的本地优先智能执行系统。

它和企业 Agent 的关系：

```text
企业 Agent：让 AI 使用订单、支付、CRM、OA、日志等企业系统。
Jarvis Lite：让 AI 使用电脑文件、桌面应用、浏览器、日历、手机通知、手表健康数据和个人知识库。
```

## 2. 当前项目进度

当前 Jarvis Lite 已具备以下能力：

- 命令行入口：支持交互式输入和 `--once` 一次性执行。
- 长期记忆：读取和写入 `memory/profile.md`，能回答用户身份问题。
- 经验记忆：通过 `/experience`、`/experiences`、`/experience-search` 和 `/experience-advice` 记录、检索和引用流程经验。
- 个人知识库：导入 Markdown、txt、PDF、JSON 聊天记录或资料目录到 `data/`，并通过 `/kb` 查看状态、通过 `/kb-summary` 查看资料摘要、通过 `/ask` 或普通问题检索回答；摘要和回答都会展示来源，并能衔接最近资料后续操作，也能用“读取项目标签资料”按标签筛出资料组继续编号读取。
- 标签系统：通过 `/tag` 给资料设置标签，标签参与知识库状态展示和检索。
- 最近上下文：保存最近资料列表、当前资料、最近文件列表、最近目录、最近搜索结果和最近建议，支持“这个资料”“第二份资料”“第一份最近文件”“这个目录”“第二条结果”等连续表达；`/read` 和“读取 note.md”等自然语言读取、按标签读取资料组、导入资料、问答命中都会更新最近资料列表，`/recent-files` 会更新最近文件列表，后续可用“读取这个资料”“读取第二份资料”“给这个资料打标签”“给第二份资料打标签”“查看第一份最近文件”“导入第一份最近文件到知识库”继续操作，日报和最近上下文状态都会引用最近上下文并生成下一步建议。
- 建议系统：经验建议会给出可执行命令，支持查看第 N 条建议。
- 执行前确认：完整建议命令可以先准备，再由“确认执行”触发；含占位符建议会返回命令草稿。
- 草稿补全：用户补全命令草稿后，会先进入确认执行状态，再由“确认执行”真正调用原命令。
- 工作台自动化：登记常用目录、查看目录、查看最近文件、按编号查看最近文件详情、按编号导入最近文件到知识库、最近文件进入最近上下文和日报、生成日报、整理目录预览、记录打开目录请求。
- 常见目录识别：未登记常用目录时，已支持自然语言识别桌面、下载目录和项目目录。
- 语音入口准备：支持 `/voice-status`、`/speak` 和 `/voice` 的第一批语音入口能力。
- 桌面入口：PySide6 小助手、助手面板、快捷命令、最近上下文/最近文件快捷入口、托盘、最近结果、主题、尺寸、置顶、透明度和开机启动。
- 更新能力：支持更新检查和更新安装包下载。
- Windows 打包：已具备 exe 图标、版本资源、安装器基础生命周期。
- 本地验证：使用 Python 3.13 和 `unittest`，最近全量测试为 271 个。

这些能力已经形成 Agent 主干：

```text
自然语言
  ↓
本地意图层
  ↓
记忆 / 知识库 / 最近上下文
  ↓
工具调用
  ↓
结果反馈
  ↓
建议和确认执行
```

## 3. 架构理解

Jarvis Lite 当前是“本地确定性 Agent”。

它暂时没有接入外部 LLM，但已经具备 Agent 的工程骨架：

```text
JarvisAgent
  ├── 本地意图解析：把自然语言映射为命令
  ├── Memory：长期记忆、经验记忆
  ├── RAG 雏形：data/ 知识库检索
  ├── Runtime Context：最近资料、最近文件、目录、搜索结果、建议
  ├── Tools：文件、目录、日报、知识库、语音、更新
  ├── Workflow：建议 -> 查看 -> 草稿 -> 准备执行 -> 确认执行
  └── Desktop UI / CLI / Voice：不同入口
```

后续接入大模型时，不应让 LLM 直接绕过本地执行层。

推荐结构：

```text
用户输入
  ↓
本地意图层优先处理
  ↓
无法确定或需要复杂规划时调用 LLM
  ↓
LLM 返回结构化意图 / 工具建议
  ↓
JarvisAgent 校验并执行
  ↓
结果返回和上下文更新
```

## 4. 三阶段设备路线

### 阶段 A：电脑 Agent

电脑是当前最适合落地的主战场。

重点能力：

- 文件和目录管理：下载目录、桌面、项目目录、资料归档。
- 个人知识库：文档导入、标签、检索、总结、问答。
- 桌面自动化：打开目录、记录操作、生成日报、快捷命令。
- 浏览器和网页资料：后续可接搜索、网页摘要、收藏整理。
- 本地任务建议：根据最近上下文给出下一步建议，并形成确认执行闭环。

本轮已完成的小任务：

- 日报建议生成：在已有最近上下文、经验和工具日志基础上，生成确定性的下一步行动建议。
- 读取资料上下文：`/read 文件名` 成功后会更新最近资料，重启后仍可继续用“这个资料”指代。
- 自然语言读取资料：“读取 note.md”“查看 note.txt”会复用 `/read`，并继承最近资料上下文更新。
- 自然语言读取最近资料：“读取这个资料”“查看这个资料”会读取当前最近资料。
- 最近资料列表：读取、导入、标签和问答命中的资料会进入最近资料列表，最近上下文和日报都能展示。
- 按编号读取最近资料：“读取第二份资料”会读取最近资料列表中的对应资料。
- 按编号给最近资料打标签：“给第二份资料打标签 项目”会给最近资料列表中的对应资料设置标签。
- 系统最近文件列表第一版：“查看最近文件”会列出常用目录、项目目录、桌面和下载目录顶层最近修改的普通文件。
- 按编号查看最近文件详情：“查看第一份最近文件”会展示最近文件的来源、路径和修改时间，不读取或打开文件。
- 按编号导入最近文件：“导入第一份最近文件到知识库”会选择最近文件列表中的对应文件，并复用 `/import` 导入 `data/`。
- 最近文件上下文联动：“查看最近上下文”和日报会展示最近文件列表，日报下一步建议会提示继续查看第一份最近文件、导入第一份最近文件到知识库或刷新 `/recent-files`。
- 最近上下文下一步建议：“查看最近上下文”会复用日报建议生成逻辑，直接提示最近资料、目录、文件和建议的后续动作。
- 桌面快捷入口补齐：面板和托盘现在可以直接触发“最近上下文”和“最近文件”。
- 知识库摘要增强：新增 `/kb-summary` 和“总结知识库”，按资料列出来源、可检索行数、标签和首行摘要预览。
- 知识库摘要上下文联动：查看摘要后可继续说“读取第一份资料”“给第二份资料打标签 项目”，重启 Agent 后仍可恢复摘要写入的资料列表。
- 知识库摘要预览截断：资料首行很长时，摘要预览会限制长度并追加 `...`，避免摘要输出被单个长段落撑开。
- 知识库摘要按标签分组：`/kb-summary` 现在会先展示标签分组，未打标签资料归入“未标签”，再保留逐份编号摘要。
- 知识库摘要按标签后续建议：有标签资料时，`/kb-summary` 会额外提示可直接复制的 `/ask 标签` 建议。
- 桌面知识库摘要快捷入口：面板和托盘新增“知识库摘要”，直接执行无参数 `/kb-summary`。
- 按标签读取知识库资料组：支持“读取项目标签资料”“查看助手标签文档”，按标签筛选资料并写入最近资料列表，后续可继续“读取第二份资料”。
- 知识库摘要按标签读取建议：`/kb-summary` 现在会直接提示“读取项目标签资料”这类按标签读取入口，和 `/ask 标签` 建议并列展示。

下一批适合做的小任务：

1. 常用目录上下文增强：桌面、下载目录和项目目录已可作为系统已知目录识别，最近资料列表、系统最近文件列表、最近文件编号详情、最近文件编号导入和最近文件上下文联动已完成第一版，后续继续补最近文件的更多操作建议。
2. 知识库摘要可读性：`/kb-summary` 已完成摘要、编号后续操作、长文本预览截断、标签分组、标签提问建议、按标签读取资料组和摘要页按标签读取建议，后续可以继续做标签组批量操作前预览或桌面摘要展示优化。
3. 跨入口任务建议：最近上下文状态已复用日报下一步建议，桌面面板和托盘已补齐最近上下文/最近文件入口，后续继续做入口细化。

### 阶段 B：手机联动

手机不应一开始追求全控制，而应从低风险同步开始。

可行方向：

- 手机通知同步：把重要提醒同步到手机。
- 日历和提醒事项：创建提醒、查看当天安排。
- 文件和照片导入：把手机资料同步到个人知识库。
- Android 自动化探索：后续根据实际设备能力接入。

落地方式可以是：

```text
Jarvis Lite 桌面端
  ↓
本地同步目录 / HTTP 小服务 / 第三方自动化入口
  ↓
手机端接收通知或数据
```

### 阶段 C：手表和健康数据

手表不是直接控制对象，更适合作为数据源。

可行方向：

- 睡眠趋势。
- 心率和运动趋势。
- 久坐提醒。
- 根据健康数据调整明日建议。

示例：

```text
读取最近睡眠数据
  ↓
读取明日日程
  ↓
判断负荷
  ↓
给出提醒或建议
```

## 5. 后续大模型接入策略

大模型接入不应替代现有本地大脑，而应作为外脑增强。

推荐职责：

- 理解复杂自然语言。
- 生成结构化意图。
- 多步骤任务规划。
- 基于资料做总结和推理。
- 生成更自然的解释和报告。

不推荐职责：

- 直接执行文件操作。
- 直接调用高风险工具。
- 绕过本地命令和测试层。
- 修改运行态规则而没有记录。

## 6. 下一步执行计划

建议执行闭环已经完成第一段：

```text
经验建议
  ↓
查看第 N 条建议
  ↓
执行第 N 条建议
  ↓
如果命令完整：准备执行
  ↓
如果缺参数：生成命令草稿
  ↓
用户补全草稿
  ↓
重新进入准备执行
  ↓
确认执行
```

常见目录识别、日报建议生成和读取资料上下文已经完成第一批。当前实现重心转向电脑 Agent 的最近文件后续操作和摘要能力：

> 最近文件后续操作与知识库摘要可读性增强。

验收标准：

- 继续保持本地确定性执行层，不让建议自动执行工具。
- 最近文件、最近资料和最近目录的状态要能被用户查看、日报引用和后续建议复用；读取过的资料和系统最近文件列表已经纳入上下文。
- 知识库摘要必须保留来源、行数、标签和可追溯预览。
- 桌面入口只接入可确认、可回溯的建议，不直接移动文件或启动高风险操作。

## 7. 长期目标

Jarvis Lite 的长期目标可以概括为：

```text
个人记忆
  + 个人知识库
  + 电脑工具
  + 手机联动
  + 手表数据
  + 大模型外脑
  = 个人设备级 Agent
```

它不是一次性做成电影里的 Jarvis，而是逐步把“自然语言 -> 工具调用 -> 上下文记忆 -> 结果反馈”这个闭环做厚。
