## 1. 对话起点

我们讨论了电影《钢铁侠》里的贾维斯式智能助手，并区分了两种目标：

- 电影版贾维斯：长期在线、深度接入现实设备、具备高度自主判断和几乎稳定的通用智能。这个目标目前不能一步到位。
- 实用版贾维斯：能语音或文本对话，理解个人偏好，调用工具做事，管理文件、知识、日程、代码和自动化流程。这个目标可以从最小版本逐步实现。

核心判断是：所有“不可能”的东西都应从最小可运行版本开始，而不是停留在想象中。

## 2. 项目愿景

做一个属于自己的个人智能助手。它不是单纯聊天机器人，而是一个能逐步接入真实工作和生活流程的本地智能中枢。

长期方向包括：

- 能和用户自然对话，理解上下文和个人表达习惯。
- 能记住长期偏好、项目背景、重要事件和常用规则。
- 能调用本机工具，帮助处理文件、代码、文档、搜索和自动化任务。
- 能根据权限边界执行动作：低风险任务自动执行，高风险任务先确认。
- 能逐步接入语音、桌面、手机、家居、车机等外部入口。

## 3. 第一阶段目标

第一阶段不追求“像电影一样聪明”，只追求“能稳定帮忙做一件真实的事”。

最小版本定义为：

- 支持命令行文本对话。
- 支持一个长期记忆文件，用来记录用户偏好、项目目标、重要事实。
- 支持读取指定目录下的文本文件，并基于文件内容回答问题。
- 支持执行少量白名单工具，例如列目录、读文件、写笔记、生成总结。
- 每次执行动作前留下日志，方便回看它做了什么。

第一阶段暂不做：

- 不做复杂 UI。
- 不做全屋设备控制。
- 不做自动发消息、自动转账、自动删除重要文件。
- 不做“自我进化”或修改底层模型。
- 不做公司项目内嵌，避免个人项目和工作项目混在一起。

## 4. 技术选型

建议第一阶段使用 Python 作为主语言。

理由：

- Python 更适合快速连接大模型、文件系统、语音、自动化脚本和本地工具。
- 个人助手早期最重要的是快速验证闭环，而不是复杂工程架构。
- 后续如果需要桌面界面，可以再接入 Web 前端、Electron、Tauri 或本地 Web UI。
- Java 适合稳定后端和企业系统，但对个人桌面自动化、AI 工具编排、语音实验来说起步较重。

初步技术结构：

```text
jarvis-lite
  ├─ README.md              项目愿景和起点记录
  ├─ memory/                长期记忆和用户偏好
  ├─ logs/                  助手执行日志
  ├─ data/                  用户提供的本地资料
  ├─ src/                   助手核心代码
  │   ├─ app.py             命令行入口
  │   ├─ agent.py           对话和任务调度
  │   ├─ memory.py          记忆读写
  │   ├─ tools.py           本地工具封装
  │   └─ config.py          配置读取
  └─ tests/                 本地测试
```

## 5. 核心模块设想

### 5.1 对话入口

第一版从命令行开始。用户输入一句话，助手返回回答。后续再扩展语音输入和语音播报。

### 5.2 记忆系统

先用本地 Markdown 或 JSON 文件保存长期记忆，不急着上数据库。

记忆内容包括：

- 用户长期目标。
- 技术偏好。
- 常用项目目录。
- 重要生活偏好。
- 助手行为规则。

### 5.3 工具系统

工具必须一开始就做边界。第一版只允许低风险工具：

- 读取指定目录文件。
- 写入项目内笔记。
- 生成总结文档。
- 列出目录结构。
- 记录操作日志。

后续再逐步加入：

- 浏览器搜索。
- 日程管理。
- 本地应用启动。
- 语音控制。
- 家居设备接口。

### 5.4 权限机制

低风险动作可以自动执行，例如读文件、生成总结、写入本项目目录。

高风险动作必须确认，例如：

- 删除文件。
- 覆盖重要文件。
- 调用外部消息接口。
- 控制设备。
- 涉及付款、账号、隐私资料的动作。

## 6. 第一版验收标准

第一版完成时，应满足以下条件：

- 可以在本机启动命令行助手。
- 可以进行多轮文本对话。
- 可以读取 `memory` 中的用户偏好并用于回答。
- 可以把一段对话总结写入本项目目录。
- 可以读取 `data` 目录中的文本资料并回答问题。
- 所有工具调用都有日志记录。
- 有最基础的本地测试，验证记忆读写和工具白名单逻辑。

## 7. 路线图

### 阶段 1：命令行助手

目标：让助手能通过命令行对话、读记忆、读文件、写总结。

### 阶段 2：个人知识库

目标：导入 Markdown、txt、PDF 摘要、聊天记录，让助手能围绕个人资料问答。

### 阶段 3：语音入口

目标：加入语音识别和语音播报，让它能像助手一样被口头调用。

### 阶段 4：桌面自动化

目标：打开程序、整理文件、生成日报、辅助写代码、管理常用目录。

### 阶段 5：外部系统接入

目标：接入日程、浏览器、消息、家居或车机等外部系统，但必须保留权限确认。

## 8. 近期下一步

下一步建议先搭建阶段 1 的最小工程：

1. 初始化 Python 项目结构。
2. 创建 `memory/profile.md` 记录用户偏好。
3. 创建命令行入口，实现一问一答。
4. 增加本地工具白名单。
5. 增加日志记录。
6. 写最小测试，确认记忆和工具调用可用。

第一阶段目标不是炫技，而是让这个助手第一次真正“动起来”。

## 9. 当前骨架运行方式

> 日期：2026-05-18
> 执行者：Codex

当前项目已经初始化 Python 命令行助手骨架，核心代码位于 `src/jarvis_lite/`，命令行入口为 `src/app.py`。

推荐使用 Python 3.13 系列创建本地虚拟环境：

```powershell
py -3.13 -m venv .venv
.\.venv\Scripts\activate
python --version
```

激活虚拟环境后，常用命令：

```powershell
python src/app.py
python src/app.py --once "/memory"
python src/app.py --once "/list"
python src/app.py --once "/ask Jarvis Lite 使用什么 Python 版本？"
python -m unittest discover -s tests -v
```

交互模式下新增会话命令：

```text
/history
/save-summary 文件名
/clear
```

当前已具备：

- 读取 `memory/profile.md` 长期记忆。
- 读取 `data/` 目录中的 `.txt` 和 `.md` 文本资料，并通过 `/ask` 或普通问题返回最多 3 条带来源的基础回答。
- 在交互式命令行中记录当前会话历史，并可把本轮会话总结写入 `word/`。
- 通过 `/memory`、`/list`、`/read`、`/ask`、`/note`、`/summary`、`/history`、`/save-summary`、`/clear` 调用第一阶段基础能力。
- 将工具调用记录到 `logs/jarvis.log`。
- 使用标准库 `unittest` 执行本地自动化测试。
- 通过 `.python-version` 和 `pyproject.toml` 固定项目使用 Python 3.13 系列。
