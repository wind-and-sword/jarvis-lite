# 标签组批量打标签待确认状态接入最近上下文计划

> 日期：2026-05-26  
> 执行者：Codex

## 目标

预览“给项目标签资料都打标签 归档”后，用户说“查看最近上下文”时能看到当前待确认的批量打标签任务、追加标签和影响资料数量；确认或取消后该状态消失。

## 设计

- 状态来源继续使用 `JarvisAgent` 内存中的 `_pending_tagged_documents_tag`、`_pending_tagged_documents_tags` 和 `_pending_tagged_documents_paths`。
- `_recent_context_status()` 增加独立的“待确认批量打标签”行。
- 空上下文判断纳入待确认批量打标签状态，避免只有待确认批量任务时误报“还没有记录”。
- 不持久化待确认批量任务，保持与当前确认闭环一致。

   - “待确认建议命令：无”仍正常展示。
   - 确认后再次查看最近上下文显示 `待确认批量打标签：无`。
