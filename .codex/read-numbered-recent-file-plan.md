# 按编号查看最近文件详情计划

> 日期：2026-05-22  
> 执行者：Codex

## 目标

用户执行“查看最近文件”后，可以继续说“查看第一份最近文件”，查看该文件的来源、路径和修改时间。

## 范围

- 保存最近文件列表到项目外 `jarvis-lite-runtime/agent-context.json`。
- 支持新 Agent 实例恢复最近文件列表。
- 支持“查看/看看/读取第 N 份最近文件”。
- 只展示元数据，不读取文件内容，不打开文件，不移动或删除文件。

## 实现步骤

1. `runtime_context.py` 增加可序列化的最近文件上下文字段。
2. `agent.py` 在 `/recent-files` 成功扫描后保存列表，并增加编号查看方法。
3. `intent.py` 增加“第 N 份最近文件”意图。
4. `tests/test_agent.py` 增加 RED/GREEN 测试：编号查看、跨实例恢复、缺列表提示。
5. 更新 README、验证记录和方案文档。

## 验收

- “查看第一份最近文件”返回文件名、来源、路径、修改时间。
- 新建 `JarvisAgent` 后仍可查看第一份最近文件。
- 无最近文件列表时提示先查看最近文件。
- 全量测试、桌面 smoke 和 `git diff --check` 通过。
