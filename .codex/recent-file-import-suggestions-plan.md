# 最近文件下一步建议补充导入动作计划

日期：2026-05-25
执行者：Codex

## 目标

按编号导入最近文件能力已经存在。下一步要让“查看最近上下文”和日报里的“下一步建议”直接提示这个动作，减少用户记忆成本。

## 方案

1. 先更新 Agent 和 Automation 测试，期望最近文件建议包含“导入第一份最近文件到知识库”。
2. 修改 `src/jarvis_lite/automation.py` 的 `suggest_next_actions_from_context()`，让最近文件建议同时包含查看、导入和刷新列表。
3. 更新 README、验证记录和阶段文档。
4. 执行专项、全量、桌面 smoke 和空白检查。

## 验收标准

- “查看最近上下文”中的最近文件建议包含“导入第一份最近文件到知识库”。
- 日报“下一步建议”中的最近文件建议包含同一动作。
- 不自动导入任何文件，不改变最近文件列表扫描和导入执行路径。
