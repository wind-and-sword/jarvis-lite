# 建议命令参数补全草稿计划

日期：2026-05-22  
执行者：Codex

## 目标

当用户对含占位符的最近建议说“执行第 N 条建议”时，Jarvis Lite 不进入待确认执行状态，而是返回可编辑命令草稿，帮助用户替换真实参数。

## 范围

- 覆盖 `/import 源文件或目录路径 [目标文件名]` 这类建议命令。
- 保持现有完整命令确认执行流程，例如 `/kb` 仍然可准备并确认执行。
- 不做真实路径推断，不自动补全参数，不持久化待确认命令。

## 实施步骤

1. 在 `tests/test_agent.py` 增加 RED 测试，断言含占位符建议返回 `命令草稿` 且确认执行仍提示无待确认命令。
2. 在 `src/jarvis_lite/agent.py` 增加命令文本抽取和占位符草稿转换。
3. 更新最近建议准备执行的占位符分支，返回草稿和参数替换提示。
4. 更新 README、进度文档、验证记录和审查记录。
5. 运行专项测试、Agent 测试、全量测试、桌面 smoke 和 `git diff --check`，通过后提交并推送。

## 验收

- `执行第一条建议` 针对 `/import 源文件或目录路径 [目标文件名]` 输出 `命令草稿：/import <源文件或目录路径> [目标文件名]`。
- 随后的 `确认执行` 仍输出没有待确认建议命令。
- 现有 `执行第二条建议` + `确认执行` 行为不变。
