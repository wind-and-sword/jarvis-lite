# 自然语言导入资料计划

> 日期：2026-05-21
> 执行者：Codex

## 上下文扫描

- `/import 源文件或目录路径 [目标文件名]` 已由 `JarvisAgent._handle_command()` 实现，支持 Markdown、txt、PDF、JSON 和目录批量导入。
- 自然语言解析集中在 `src/jarvis_lite/intent.py`，已有命令映射模式可直接返回 `/import ...`。
- 当前自然语言进度文档的后续建议包含“导入这个文件到知识库”，但“这个文件”需要上下文指代。本阶段先做明确路径，不做指代。

## 方案

- 支持明确路径表达：
  - `导入 C:\path\note.md 到知识库`
  - `把 C:\path\note.md 导入知识库`
  - 路径带空格时允许用户加引号。
- 解析后复用 `/import "源路径"`，继续由现有导入逻辑处理文件类型、重名、错误提示和知识库索引。
- 不新增文件选择器、拖拽入口、最近文件上下文或模糊搜索。

## 验收

- RED：自然语言导入表达先无法导入资料。
- GREEN：自然语言导入表达能把外部资料导入 `data/`，并可被 `/ask` 检索。
- 验证：新增 Agent 测试、`tests.test_agent`、全量测试、桌面 smoke、`git diff --check`。
