# 最近上下文输出下一步建议计划

> 日期：2026-05-22  
> 执行者：Codex

## 目标

把日报中的确定性“下一步建议”复用到“查看最近上下文”，形成跨入口任务建议的第一步。

## 范围

- 抽取公共建议生成函数，不解析日报 Markdown。
- “查看最近上下文”展示“下一步建议”段。
- 建议覆盖最近资料、最近目录、最近文件和最近建议。
- 不自动执行建议，不新增确认执行规则。
- 不读取系统最近文件内容，不打开文件。

## 实现步骤

1. 在 `tests/test_agent.py` 添加 RED 测试，证明最近上下文缺少下一步建议。
2. 在 `automation.py` 抽取 `suggest_next_actions_from_context()`，让日报继续复用。
3. 在 `agent.py` 构造当前 `RuntimeContext`，并在最近上下文状态中追加建议。
4. 运行 Agent、Automation 专项和全量验证。
5. 更新 README、验证记录、进度文档和方案文档。

## 验收

- “查看最近上下文”输出“下一步建议：”。
- 有最近资料时提示 `/read` 与 `/tag`。
- 有最近目录时提示 `/organize-preview` 与 `/dir-open`。
- 有最近文件时提示“查看第一份最近文件”和 `/recent-files`。
- 有最近建议时提示“查看第一条建议”和“执行第一条建议”。
