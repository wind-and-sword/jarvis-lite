# 最近搜索结果编号选择计划

> 日期：2026-05-21
> 执行者：Codex

## 上下文扫描

- 最近搜索结果上下文已经把 `/ask` 或普通问题的第一条命中资料记录为最近资料。
- `answer_from_matches()` 输出带编号的结果列表，例如 `1. 根据 data/...`。
- 用户继续操作结果时，真实需求可能不是第一条，而是“第二条结果”。
- 当前 Agent 没有保存最近一次问答的完整结果列表，只保存第一条资料。

## 范围

- Agent 保存最近一次问答返回的资料路径列表。
- 支持“给第二条结果打标签 长期记忆”“给第2条结果打标签 运行环境”。
- 支持没有最近搜索结果时给出明确提示。
- 支持序号超出范围时给出当前结果数量提示。

## 不做

- 不改变搜索排序、过滤和回答格式。
- 不做“第二条结果是什么”这类阅读型追问。
- 不跨会话持久化最近结果列表。

## 验收

- RED：问答返回两条资料后，“给第二条结果打标签”先无法选择第二条。
- RED：没有最近搜索结果时，“给第二条结果打标签”先没有专门提示。
- GREEN：第二条结果能被正确打标签。
- 全量 `unittest`、桌面 smoke、`git diff --check` 通过。
