# 自然语言最近资料和最近目录持久化计划

> 日期：2026-05-21
> 执行者：Codex

## 目标

让最近资料和最近目录像最近搜索结果一样写入项目外运行态上下文。重启或新建 `JarvisAgent` 后，用户仍可继续说“给这个资料打标签 项目”或“打开这个目录”。

## 范围

- `RuntimeContext` 增加最近资料相对路径字段。
- `RuntimeContext` 增加最近目录字段，保存别名和绝对路径字符串。
- `JarvisAgent` 初始化时恢复最近资料和最近目录。
- 资料、搜索结果、目录状态变化时统一保存完整运行态上下文。

## 非目标

- 不做跨设备同步。
- 不引入数据库、缓存服务或新依赖。
- 不改变资料检索排序、文件整理预览、目录打开记录格式。
- 不实际启动资源管理器或移动文件。

## 验收

- 导入单个资料后新建 Agent，`给这个资料打标签 项目` 能更新刚导入资料。
- 打开常用目录后新建 Agent，`打开这个目录` 能复用刚打开目录。
- 损坏或缺失运行态文件仍为空上下文。
- `tests.test_agent` 专项通过，全量 `unittest discover`、桌面 smoke 和 `git diff --check` 通过。
