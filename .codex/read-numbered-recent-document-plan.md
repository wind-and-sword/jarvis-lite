# 按编号读取最近资料计划

> 日期：2026-05-22  
> 执行者：Codex

## 目标

用户在“查看最近上下文”看到最近资料列表后，可以继续说“读取第二份资料”来读取列表中的对应资料。

## 方案

- `intent.py` 增加最近资料编号读取解析，识别“读取/查看/看看第 N 份资料/文档/文件”。
- `agent.py` 增加 `read_numbered_recent_document` 路由和读取方法。
- 读取成功时复用 `/read`，让当前资料和最近资料列表继续沿用已有更新逻辑。
- 缺少列表和编号越界时返回明确中文提示。

## 验收

- RED：新增 Agent 测试先证明“读取第二份资料”不能读取最近资料列表。
- GREEN：读取第二份资料返回对应 data 文件内容，并带出文件路径。
- GREEN：缺少最近资料列表时提示先读取或导入。
- 回归：“查看第二条结果”仍然走最近搜索结果路径。
- 收尾：专项测试、全量测试、桌面 smoke 和 `git diff --check` 通过。
