# 标签组批量打标签前预览计划

> 日期：2026-05-26
> 执行者：Codex

## 目标

让用户可以说“给项目标签资料都打标签 归档”，先看到会影响哪些资料和建议逐份确认命令，而不是直接批量修改知识库标签。

## 实现设计

- 在自然语言意图层新增 `preview_tagged_documents_tagging`。
- 标签组名放入 `alias`，拟追加的新标签放入 `tags`。
- 解析顺序放在普通 `_parse_tag_intent()` 之前，避免把“项目标签资料”当成文件名。
- Agent 复用 `build_knowledge_index()` 按现有标签筛选资料。
- 预览输出：
  - 标签组名和拟追加标签。
  - 每份匹配资料的路径、当前标签和预览标签。
  - 明确说明不会修改资料标签。
  - 写入最近资料列表，给出逐份确认命令示例。
- 标签组不存在时返回明确提示并建议 `/kb-summary`。

## 验收标准

- RED/GREEN 覆盖成功预览并保持标签不变。
- RED/GREEN 覆盖标签组不存在提示。
- Agent、Knowledge、全量测试、桌面 smoke 和 `git diff --check` 通过。
