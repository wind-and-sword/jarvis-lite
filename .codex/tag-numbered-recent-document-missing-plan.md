# 编号最近资料打标签缺失提示计划

日期：2026-05-27  
执行者：Codex

## 目标

用户说“给第二份资料打标签 项目”时，如果最近资料列表里的对应文件已经删除或移动，直接保留编号上下文并提示资料缺失。

## 设计

- 不修改最近资料运行态结构。
- 在 `_tag_numbered_recent_document()` 调用 `/tag` 前检查文件是否存在。
- 缺失时返回 `第 N 份资料：data/<路径>（资料缺失）`，并提示查看 `/kb` 或重新导入资料。
- 正常打标签、空列表、越界提示保持不变。

## 步骤

1. 在 `tests/test_agent.py` 新增失败测试，覆盖给已删除的第二份最近资料打标签。
2. 运行目标测试确认 RED。
3. 修改 `agent.py` 的编号打标签路径，补文件存在性判断。
4. 运行目标测试和相邻回归。
5. 更新 README、进度文档、验证记录和审查记录。
6. 全量验证后提交并推送。

## 验收

- “给第二份资料打标签 项目”对缺失文件返回编号缺失提示。
- 正常“给第二份资料打标签 项目”仍能更新标签。
- 编号读取缺失提示、批量标签历史资料缺失提示和普通标签回归通过。
