# 按编号给最近资料打标签计划

> 日期：2026-05-22  
> 执行者：Codex

## 目标

用户在“查看最近上下文”看到最近资料列表后，可以继续说“给第二份资料打标签 项目”。

## 方案

- `intent.py` 在标签解析中识别“第 N 份资料/文档/文件”，返回 `tag_numbered_recent_document`。
- `agent.py` 增加该意图路由，按编号从最近资料列表选择文件。
- 复用现有 `/tag` 命令，保证标签写入、日志和最近资料更新逻辑一致。
- 保留“给第二条结果打标签”的最近搜索结果语义。

## 验收

- RED：新增测试先证明“给第二份资料打标签 项目”会误走普通 `/tag` 文件名。
- GREEN：按编号给最近资料打标签成功。
- GREEN：缺少最近资料列表时提示先读取或导入。
- 回归：给第二条搜索结果打标签仍然可用。
- 收尾：专项测试、全量测试、桌面 smoke 和 `git diff --check` 通过。
