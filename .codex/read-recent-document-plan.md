# 自然语言读取最近资料计划

> 日期：2026-05-22  
> 执行者：Codex

## 目标

用户已经有最近资料上下文后，可以说“读取这个资料”“查看这个资料”读取当前资料内容。

## 范围

- 新增自然语言意图 `read_recent_document`。
- 复用 `JarvisAgent._recent_document_path` 和现有 `/read` 命令。
- 没有最近资料时返回明确提示。
- 不新增运行态字段，不改变 `/read` 输出格式，不改变编号搜索结果和编号建议读取。

## 实施步骤

1. 增加 RED 测试：自然语言读取具体资料后，再说“读取这个资料”应返回同一文件内容。
2. 增加 RED 测试：无最近资料时，“读取这个资料”应提示缺少最近资料。
3. 在 `intent.py` 中识别“读取/查看/看看 这个资料/当前资料/刚才的资料/最近的资料”。
4. 在 `agent.py` 中新增处理分支，复用 `/read`。
5. 跑专项和完整验证，更新文档与留痕。

## 验收标准

- “读取这个资料”可读取最近资料内容。
- 无最近资料时不误读文件名。
- “查看第一条结果”和“查看第一条建议”保持原行为。
