# 最近搜索结果上下文计划

> 日期：2026-05-21
> 执行者：Codex

## 上下文扫描

- `/ask` 和普通资料问答都通过 `answer_from_data()` 返回文本答案，当前不会把命中的资料写入最近上下文。
- `JarvisAgent` 已有 `_recent_document_path`，可复用为“最近资料/最近结果”的目标资料路径。
- `knowledge.search_data()` 已返回结构化 `DataMatch`，包含 `relative_path`、`line_number`、`text` 和 `score`，适合避免从回答文案里反向解析来源。
- `intent.py` 已支持“这个资料/最近的资料”打标签，本轮扩展“这个结果/最近的结果”等搜索结果指代。

## 范围

- `/ask 问题` 命中资料后，记录第一条命中资料为最近资料。
- 普通自然语言问题命中资料后，也记录第一条命中资料为最近资料。
- 用户可说“给这个结果打标签 运行环境”，复用现有 `/tag` 流程。
- 无搜索命中时不更新最近资料。

## 不做

- 不改变检索排序、得分或回答格式。
- 不做编号选择，例如“给第二条结果打标签”。
- 不跨会话持久化最近搜索结果。

## 验收

- RED：`/ask` 命中后，“这个结果”无法被打标签。
- RED：普通资料问答命中后，“这个结果”无法被打标签。
- GREEN：两种问答入口都能把第一条命中的资料记录为最近资料。
- 全量 `unittest`、桌面 smoke、`git diff --check` 通过。
