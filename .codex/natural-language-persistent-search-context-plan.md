# 最近搜索结果持久化计划

> 日期：2026-05-21
> 执行者：Codex

## 上下文扫描

- 最近搜索结果列表当前保存在 `JarvisAgent._recent_search_result_paths`。
- “查看第二条结果”和“给第二条结果打标签”都依赖该列表。
- 桌面设置和更新下载已经使用项目外 `jarvis-lite-runtime` 目录保存运行态文件。
- 当前缺口是新建 Agent 实例后上下文丢失。

## 范围

- 将最近搜索结果路径列表写入 `jarvis-lite-runtime/agent-context.json`。
- Agent 初始化时读取该文件并恢复最近搜索结果列表。
- 若文件缺失、损坏或字段类型不正确，回退为空上下文。
- 本阶段只持久化最近搜索结果列表。

## 不做

- 不持久化最近目录。
- 不持久化最近导入资料。
- 不改变知识库检索排序、过滤和回答格式。
- 不引入数据库或第三方依赖。

## 验收

- RED：第一个 Agent 问答后，新 Agent 说“查看第二条结果”先失败。
- GREEN：新 Agent 可以恢复最近结果列表并查看第二条结果。
- Agent 专项测试通过。
- 全量 `unittest`、桌面 smoke、`git diff --check` 通过。
