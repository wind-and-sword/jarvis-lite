# 系统最近文件列表第一版计划

> 日期：2026-05-22  
> 执行者：Codex

## 目标

让用户可以说“查看最近文件”或使用 `/recent-files` 查看电脑工作台里最近修改的文件，作为电脑 Agent 最近文件列表的第一步。

## 范围

- 扫描已登记常用目录，以及已知目录：项目目录、桌面、下载目录。
- 只扫描目录顶层普通文件，不递归，不读取文件内容。
- 按修改时间倒序返回最多 5 个文件。
- 不打开、不移动、不删除文件。
- 不改变最近资料列表和最近搜索结果语义。

## 实现步骤

1. 在 `automation.py` 增加 `RecentFile` 数据结构和 `list_recent_files()`。
2. 在 `intent.py` 增加“查看最近文件”“最近文件”等自然语言映射。
3. 在 `agent.py` 增加 `/recent-files` 命令和自然语言路由，复用常用目录与已知目录发现逻辑。
4. 增加 Agent 和 Automation 测试，先验证 RED，再实现 GREEN。
5. 更新 README、验证记录和个人设备级 Agent 方案。

## 验收

- “查看最近文件”会展示最近文件列表，并按修改时间倒序。
- `/recent-files` 在没有可用文件时给出明确提示。
- 子目录不会作为最近文件输出。
- 全量测试、桌面 smoke 和 `git diff --check` 通过。
