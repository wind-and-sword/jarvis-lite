# 已知下载目录上下文增强计划

日期：2026-05-22  
执行者：Codex

## 目标

让 Jarvis Lite 在未登记常用目录时，也能识别本机常见下载目录，使“打开下载目录”和“整理下载目录”进入现有目录工作流。

## 范围

- 支持中文别名：`下载`、`下载目录`、`下载文件夹`。
- 支持英文别名：`download`、`downloads`。
- 候选目录名：`Downloads`、`下载`。
- 复用现有目录打开记录和整理预览逻辑。
- 不真实启动资源管理器，不移动或删除文件。

## 实施步骤

1. 新增 RED 测试：模拟 `Path.home()` 下存在 `Downloads`，验证“打开下载目录”记录打开请求。
2. 新增 RED 测试：模拟 `Path.home()` 下存在 `Downloads`，验证“整理下载目录”返回整理预览并标记为下载目录。
3. 扩展 `JarvisAgent._known_directory_candidates()`。
4. 更新 README、个人设备级 Agent 方案、自然语言进度、验证记录。
5. 执行专项测试、全量测试、桌面 smoke 和 `git diff --check`。

## 验收

- “打开下载目录”返回 `已记录打开目录请求：下载`。
- “整理下载目录”返回 `文件整理预览：下载`，且只生成预览。
- 现有“打开桌面”“整理桌面”行为不变。
